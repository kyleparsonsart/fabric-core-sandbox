Fabric · Design Systems

Token Adoption

How much of Apollo's Figma work is powered by Fabric color, typography, and spacing tokens — and at which tier.

A token reference is a unique Figma variable subscribed to across a team's files — not a per-instance count, but a signal that a file is connected to that token. Tracked Fabric collections: Semantics.Color, Primitives.Color (+ v2), Semantics.Typography, Primitives.Typography, Semantics.Spacing, Primitives.Spacing.

Total Token References
—

All distinct Figma variable subscriptions across every team — Fabric and non-Fabric combined. One reference = one file is subscribed to one token. This is the full denominator the adoption rate is measured against.

Fabric Adoption Rate
—

Share of all token references that use a Fabric-defined collection (semantic or primitive tier). The remainder are local, custom, or third-party tokens that exist outside the shared system.

Semantic Tier
—

References to Semantics.* collections — the recommended, future-proof tier. Semantic tokens express design intent ("color.text.primary") not raw values, so a single system change cascades everywhere. Should trend up over time.

Primitive Tier
—

References to Primitives.* collections — raw values like hex codes or pixel sizes. These are the building blocks of Fabric but not the recommended usage tier. Should trend down as teams migrate to semantic tokens.
